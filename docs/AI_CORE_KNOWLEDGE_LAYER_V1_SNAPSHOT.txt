SECTION 01: CORE PURPOSE
=======================

목적:
- AI 지식 기반을 단일 그래프 형태로 통합한다.
- HANJA, Bible KOR, Bible KJV, Strong, Greek, Hebrew의 지식을 상호 연결한다.
- 단일 조회 성능과 확장 가능한 지식 연결을 동시에 확보한다.

SECTION 03: SOURCE TABLE
======================

TABLE: ai_knowledge_sources

설명:
- 데이터 원본(소스) 카탈로그 관리

허용 sourceCode:
- HANJA
- BIBLE_KOR
- BIBLE_KJV
- STRONG_HEBREW
- STRONG_GREEK
- GREEK
- HEBREW

SECTION 04: ENTRY TABLE
=====================

TABLE: ai_knowledge_entries

설명:
- 통합 지식 엔트리 마스터

허용 entryType:
- HANJA_CHAR
- BIBLE_VERSE
- STRONG_NUMBER
- GREEK_WORD
- HEBREW_WORD
- CONCEPT
- TOPIC

SECTION 05: TOKEN TABLE
=====================

TABLE: ai_knowledge_tokens

설명:
- 통합 검색 토큰 저장소

허용 tokenType:
- hanja
- korean
- english
- bible_ref
- strong
- greek
- hebrew
- unicode
- radical
- meaning
- sound
- topic

SECTION 06: RELATION TABLE
========================

TABLE: ai_knowledge_relations

설명:
- 지식 엔트리 간 연결 그래프

허용 relationType:
- MEANS
- READS_AS
- APPEARS_IN
- TRANSLATES_TO
- STRONG_LINK
- ORIGINAL_WORD
- CROSS_REFERENCE
- TOPIC_RELATED
- SEMANTIC_RELATED

SECTION 07: TAG TABLE
===================

TABLE: ai_knowledge_tags

설명:
- 주제 분류/태그 사전

예시:

- 구원
- 믿음
- 복음
- 성전
- 집
- 말씀

SECTION 08: ENTRY TAG TABLE
========================

TABLE: ai_knowledge_entry_tags

설명:
- ai_knowledge_entries와 ai_knowledge_tags의 다대다 연결 테이블

- datetime
