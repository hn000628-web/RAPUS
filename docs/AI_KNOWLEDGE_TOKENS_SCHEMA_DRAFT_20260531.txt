==================================================
1) 목적
==================================================
- Codex/LLM 추론 성능 향상을 위한 검색/정규화 토큰 인덱스 계층 정의.
- Bible/Hanja/Theology/Strong/Greek/Hebrew 확장 가능한 공통 토큰 구조 설계.
- 본 문서는 DDL 초안 문서이며, DB 실행/코드 반영은 포함하지 않음.

==================================================
2) 현재 분석 결론
==================================================
- ai_bible_verses: 원문 저장 계층(완료/운영 기준).
- ai_interpretation_links: 엔트리 간 관계 그래프 계층.
- ai_library_embeddings: 벡터/모델 산출물 메타 계층.
- ai_knowledge_tokens(신규 후보): 검색/정규화 토큰 전용 계층이 필요.
- 원문을 토큰 테이블에 중복 저장하면 역할 충돌/비용 증가 발생.

==================================================
3) 테이블 역할
==================================================
Table: ai_knowledge_tokens
Role:
- target 엔트리(예: Bible verse)에 대해 검색 가능한 토큰만 저장.
- token/normalizedToken 기반 exact/prefix/filter 검색 지원.
- canonicalKey/sourceId 기반 소스 교차 검색 성능 보강.

-- DESIGN DRAFT ONLY (DO NOT EXECUTE HERE)
CREATE TABLE ai_knowledge_tokens (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  target_type TEXT NOT NULL,
  target_id BIGINT NOT NULL,
  source_id BIGINT,
  canonical_key TEXT,
  token_type TEXT NOT NULL CHECK (
    token_type IN (
      'bible_ref','korean','english','hanja','topic','strong','greek','hebrew'
    )
  ),
  token TEXT NOT NULL,
  normalized_token TEXT NOT NULL,
  position INTEGER,
  weight NUMERIC(10,4) NOT NULL DEFAULT 1.0000,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

==================================================
9) 금지사항
==================================================
- 원문 전체(verseText/bodyText) 저장 금지.
- 대형 JSON payload 저장 금지.
- 임베딩 벡터 본문 저장 금지(embeddings 계층 역할 침범 금지).
- 관계 그래프 정보를 토큰 테이블로 중복 저장 금지.

==================================================
10) ai_bible_verses 연결 방식
==================================================
- targetType='BIBLE_VERSE'
- targetId=ai_bible_verses.id
- sourceId=ai_bible_verses.sourceId
- canonicalKey=ai_bible_verses.canonicalKey

예시
- GEN.1.1(KOR/KH/KJV) 각각 targetId는 다르나 canonicalKey는 동일.
- token 검색 후 canonicalKey로 다국어 교차 조회 가능.

==================================================
11) ai_interpretation_links와 역할 분리
==================================================
- ai_knowledge_tokens: "검색 단위" 저장.
- ai_interpretation_links: "관계(edge)" 저장.
- 토큰 검색 결과를 relation으로 확장하는 순차 구조를 권장.

==================================================
12) ai_library_embeddings와 역할 분리
==================================================
- ai_knowledge_tokens: lexical / rule-based retrieval.
- ai_library_embeddings: semantic / vector retrieval 메타.
- 하이브리드 검색: token recall -> embedding rerank.

==================================================
13) 향후 ETL 흐름
==================================================
STEP 01
- ai_bible_verses 로드

STEP 02
- verseText 정규화
- token 분해 (korean/english/bible_ref 등)

STEP 03
- ai_knowledge_tokens bulk upsert

STEP 04
- canonicalKey 기준 소스 교차 검증

STEP 05
- ai_interpretation_links 생성(translation_pair 등)

STEP 06
- ai_library_embeddings 메타 연결
