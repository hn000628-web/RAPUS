==================================================
SECTION 01 : CORE PRINCIPLE
==================================================

users.corporationGrade
= 회사 직급권한 정규 기준

users.genesisGrade
= 플랫폼 최상위 보안등급 / Genesis 보호 계층

users.userGrade
= legacy / deprecated account grade marker

profiles.operationGrade
= 프로필/채널 운영 capability

profiles.governanceTier
profiles.governanceSymbol
profiles.governanceMetadata
= symbolic governance metadata only

실제 security / auth / payment / admin capability와
symbolic governance layer는 절대 혼합하지 않는다.

==================================================
SECTION 02 : USERS GRADE / CORPORATION GRADE / GENESIS GRADE
==================================================

users.corporationGrade:

0 = 수습사원
1 = 팀원
2 = 대리
3 = 팀장
4 = 과장
5 = 상무
6~11 = 확장 직급권한
12 = 최종 OWNER_ACCOUNT / 최고직급권한

users.userGrade:

0~11 = 실제 계정 권한 등급
12 = 최종 OWNER_ACCOUNT

users.genesisGrade:

0 = NONE
1~11 = RESERVED
12 = GENESIS_OWNER

DB:

userGrade INTEGER NOT NULL DEFAULT 0
CHECK(userGrade BETWEEN 0 AND 12)

corporationGrade INTEGER NOT NULL DEFAULT 0
CHECK(corporationGrade BETWEEN 0 AND 12)

genesisGrade INTEGER NOT NULL DEFAULT 0
CHECK(genesisGrade BETWEEN 0 AND 12)

주의:

users.userGrade는 즉시 삭제하지 않고 deprecated로 유지한다.
users.genesisGrade는 corporationGrade보다 상위의 플랫폼 최상위 보호 계층이다.
실제 계정/회사/보호/운영 기준은 users.userGrade / users.corporationGrade / users.genesisGrade / profiles.operationGrade를 구분해 사용한다.
profiles.governanceTier의 12는 symbolic layer 값이며 auth/security/payment 권한으로 연결하지 않는다.

SECTION : CORPORATION GRADE AUTHORITY RULE
==================================================

users.corporationGrade는 회사 직급권한의 정규 컬럼이다.

권한값:
0 = 수습사원
1 = 팀원
2 = 대리
3 = 팀장
4 = 과장
5 = 상무
6~11 = 확장 직급권한
12 = 최종 OWNER_ACCOUNT / 최고직급권한

DB:

corporationGrade INTEGER NOT NULL DEFAULT 0
CHECK(corporationGrade BETWEEN 0 AND 12)

genesisGrade INTEGER NOT NULL DEFAULT 0
CHECK(genesisGrade BETWEEN 0 AND 12)

정책:
- 기존 users.userGrade는 삭제하지 않고 deprecated로 유지한다.
- 기존 userGrade 값은 corporationGrade로 백필한다.
- users.genesisGrade는 corporationGrade보다 상위의 플랫폼 최상위 보호 계층이다.
- OWNER_ACCOUNT hn0628@naver.com 은 userGrade = 12 / corporationGrade = 12 / genesisGrade = 12 를 적용한다.
- test1/test2 계정은 corporationGrade = 0 을 유지한다.
- profiles.operationGrade는 프로필/채널 운영 capability 기준으로 유지한다.
- profiles.governanceTier는 symbolic 전용이며 실제 권한으로 사용하지 않는다.
- 실제 권한 판정은 users.userGrade / users.corporationGrade / users.genesisGrade / profiles.operationGrade를 역할별로 구분해 사용한다.
