AI LIBRARY SQLITE ONLY SYSTEM SNAPSHOT

SECTION 01 : CORE CONCEPT
AI 한국어 해석 증진을 위한 전용 지식 DB를 운영 DB와 분리한다.

SECTION 02 : SQLITE ONLY RULE
AI 라이브러리는 SQLite 전용으로 관리한다.
운영 DB 스키마와 분리하여 독립 init 파일로 생성한다.

SECTION 03 : SEPARATE DB FILE RULE
AI 전용 DB 파일은 backend/data/ai-library.sqlite 로 고정한다.
운영 DB(prod.sqlite)와 직접 결합하지 않는다.

SECTION 04 : PRIMARY SOURCE RULE
우선순위 1순위는 사용자 제공 자료이다.
sourceType으로 자료 계층을 분리하고 sourceCode로 원천을 식별한다.

SECTION 05 : BIBLE DATA RULE
성경 데이터는 ai_bible_verses에 저장한다.
KO/KH/KJV를 병렬 구조로 저장하며 UNIQUE(sourceId, verseKey)를 유지한다.

SECTION 06 : HANJA DATA RULE
한자 데이터는 ai_hanja_dictionary에 저장한다.
hanjaWord, koReading 인덱스로 해석 보강 탐색을 지원한다.

SECTION 07 : THEOLOGY SNAPSHOT RULE
신학/교리 스냅샷은 ai_theology_snapshots에 저장한다.
doctrineType, stanceType 기준으로 표준/경고/참조를 구분한다.

SECTION 08 : KOREAN INTERPRETATION RULE
한국어 해석 용어는 ai_korean_interpretation_terms를 우선 참조한다.
termType으로 성경어휘/한자어/신학어/존재론 표현을 분리한다.

SECTION 09 : EMBEDDING KEY RULE
임베딩은 ai_library_embeddings에 embeddingKey만 저장한다.
SQLite에 vector binary를 직접 저장하지 않는다.

SECTION 10 : SERVICE DB ISOLATION RULE
users/profiles/products/cart/orders 등 서비스 DB와 FK 연결을 금지한다.
AI 라이브러리 스키마는 독립 운영한다.

SECTION 11 : FINAL DECLARATION
AI LIBRARY는 운영 DB와 분리된 SQLite 전용 지식 원장이다.
해석 기준은 한국어 용어, 성경 구절, 한자 사전, 신학 스냅샷 순으로 통합 참조한다.
