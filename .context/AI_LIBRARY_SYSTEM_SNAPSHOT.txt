AI LIBRARY SYSTEM SNAPSHOT

==================================================
SECTION 01 : CORE CONCEPT
==================================================

AI LIBRARY SYSTEM은 일반 서비스 도메인과 분리된
AI 지식 원장 도메인이다.

대상:
- 성경
- 한자
- 국어사전

통합 저장 원칙:
- sourceType 기반 단일 엔트리 구조
- 원문/용어/주석/임베딩 키 분리

==================================================
SECTION 02 : SOURCE RULE
==================================================

테이블: ai_library_sources

sourceType:
- BIBLE
- HANJA
- KOREAN_DICTIONARY

역할:
- 데이터 출처 단위 메타 관리
- sourceCode 고유 식별

==================================================
SECTION 03 : ENTRY RULE
==================================================

테이블: ai_library_entries

역할:
- 실제 원문/정규화/번역/사전 분류 저장
- sourceId + entryCode 유니크
- sourceType으로 도메인 분기

==================================================
SECTION 04 : TERM SEARCH RULE
==================================================

테이블: ai_library_terms

역할:
- 검색어/표제어/동의어/발음/태그 분리 저장

termType:
- HEADWORD
- KEYWORD
- SYNONYM
- PRONUNCIATION
- TAG

normalizedTerm 인덱스로 검색 최적화

==================================================
SECTION 05 : ANNOTATION RULE
==================================================

테이블: ai_library_annotations

annotationType:
- COMMENTARY
- INTERPRETATION
- MEMO
- CROSS_REFERENCE

visibility:
- PRIVATE
- INTERNAL
- PUBLIC

주석은 entryId 기준으로 별도 관리한다.

==================================================
SECTION 06 : EMBEDDING RULE
==================================================

테이블: ai_library_embeddings

저장 정책:
- 벡터 binary 직접 저장 금지
- embeddingKey만 저장
- provider/model/dimension 메타 보관

향후 PostgreSQL + pgvector 전환 가능 구조 유지

==================================================
SECTION 07 : BIBLE STRUCTURE RULE
==================================================

성경 데이터는 sourceType='BIBLE'로 저장한다.
chapterCode / verseNumber 컬럼으로 장절 구조를 표현한다.

==================================================
SECTION 08 : HANJA STRUCTURE RULE
==================================================

한자 데이터는 sourceType='HANJA'로 저장한다.
strokeCount / radical 컬럼으로 한자 속성 표현 가능하다.

==================================================
SECTION 09 : KOREAN DICTIONARY STRUCTURE RULE
==================================================

국어사전 데이터는 sourceType='KOREAN_DICTIONARY'로 저장한다.
dictionaryCategory와 terms로 의미/용례/검색 확장 가능하다.

==================================================
SECTION 10 : AI USAGE RULE
==================================================

Frontend 직접 DB 접근 금지.
DB 접근은 Service Layer만 허용.

AI 추천/해석/검색은
sources -> entries -> terms/annotations/embeddings 조합으로 수행한다.

==================================================
SECTION 11 : FINAL DECLARATION
==================================================

AI LIBRARY SYSTEM은
성경/한자/국어사전을 통합 수용하는
플랫폼 AI 지식 원장 공식 구조이다.
