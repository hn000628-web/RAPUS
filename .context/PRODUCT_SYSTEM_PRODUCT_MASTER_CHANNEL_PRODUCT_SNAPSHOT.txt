PRODUCT SYSTEM SNAPSHOT

MASTER PRODUCTS
+
CHANNEL PRODUCTS

공식 구조

==================================================
SECTION 01 : CORE CONCEPT
==================================================

PRODUCT SYSTEM은 다음 2계층으로 고정한다.

1) master_products
- 공산품 전역 상품원장
- 전국 공용 기준
- 바코드/표준 상품 기준

2) channel_products
- 채널별 판매상품 원장
- 음식점 메뉴, 카페 메뉴, 마트 판매상품, 채널 전용상품 포함
- 주문/장바구니 연결 기준

==================================================
SECTION 02 : MASTER PRODUCTS RULE
==================================================

master_products 규칙:
- 공산품 전용 전역 원장
- channelCode 금지
- 주문 직접 연결 금지
- productCode UNIQUE
- productStatus: ACTIVE / INACTIVE / DELETED
- approvalStatus: DRAFT / AUTO_IMPORTED / APPROVED / REJECTED

==================================================
SECTION 03 : CHANNEL PRODUCTS RULE
==================================================

channel_products 규칙:
- channelCode 필수
- UNIQUE(channelCode, productCode)
- productSourceType:
  MASTER_PRODUCT / CHANNEL_MENU
- MASTER_PRODUCT -> masterProductId 필수
- CHANNEL_MENU -> masterProductId NULL
- 주문/장바구니/판매 상태 기준 테이블

==================================================
SECTION 04 : PRODUCTCODE RULE
==================================================

공산품:
- 실제 바코드 사용
- 예: 8801043014830 / 8801116000012 / 8801007000001

자체 메뉴:
- MENU_{channelCode}_{sequence}
- 예: MENU_B012712392766_000001

==================================================
SECTION 05 : ORDER CONNECTION RULE
==================================================

절대 규칙:
- 주문은 master_products 기준 금지
- 주문은 channel_products 기준

연결 원칙:
- cart_items -> channelProductId
- order_items -> channelProductId
- masterProductId 직접 주문 연결 금지

==================================================
SECTION 06 : CHANNELCODE RULE
==================================================

channelCode 단일 귀속:
- channel_products -> channelCode 필수
- master_products -> channelCode 금지

금지:
- ownerChannelCode 생성 금지
- rootChannelCode 생성 금지
- profileChannelCode 생성 금지

==================================================
SECTION 07 : DB RELATION RULE
==================================================

관계:
- master_products (1) -> channel_products (N)
- channel_products.masterProductId -> master_products.id

단, sourceType이 CHANNEL_MENU인 경우
- masterProductId는 NULL 허용

==================================================
SECTION 08 : FUTURE CART RULE
==================================================

장바구니는 channel_products 기준으로만 구성한다.
- 채널별 가격/재고/행사 상태를 직접 반영
- 동일 master 상품도 채널별 판매조건 분리

==================================================
SECTION 09 : FUTURE ORDER RULE
==================================================

주문은 channel_products 판매 스냅샷을 기준으로 확정한다.
- 주문 시점 가격/재고/행사 상태 고정
- 채널 운영정책(배달/픽업/이벤트)과 동기화

==================================================
SECTION 10 : FINAL DECLARATION
==================================================

PRODUCT SYSTEM 공식 선언:

master_products
= 공산품 전역 상품원장

channel_products
= 채널 판매상품 원장

주문/장바구니 기준
= channel_products

master_products direct order 연결
= 금지
