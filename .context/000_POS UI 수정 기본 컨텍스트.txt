POS UI 수정 작업은 코드 확인보다 실제 웹 UI 확인을 우선한다.

기본 진입 플로우:

1. http://localhost:3000/login 접속
2. test2@prod.com / 1234 로그인
3. 로그인 후 마이페이지 진입
4. 마이페이지의 [포스] 버튼 또는 메뉴 클릭
5. 실제 POS 화면 진입
6. 브라우저에서 UI 문제를 먼저 확인
7. 확인된 문제만 관련 코드에서 최소 수정
8. 수정 후 동일 플로우로 다시 브라우저 확인

절대 원칙:

- 코드 먼저 추정 수정 금지
- 실제 UI에서 확인되지 않은 문제 임의 수정 금지
- POS UI 수정은 반드시 웹 로그인 후 화면 확인 기준으로 진행
- DB / API / Service / Controller 수정 금지
- init-db.ts 수정 금지
- prod.sqlite 삭제 금지
- 테스트 계정 데이터 임의 변경 금지

- DB 스키마 변경
- API 추가/변경
- Service/Controller 변경

- 신규 테이블 생성
- 기존 정상 라우트 삭제
- 공통 레이아웃 대규모 재작성
