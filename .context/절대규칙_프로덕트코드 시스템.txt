==================================================
SECTION 00 : FINAL CORE PRINCIPLE
==================================================

RAPUS 상품 시스템 핵심 철학:

현실 유통 코드를
RAPUS semantic platform language로 변환한다.

즉:

현실 바코드
+
semantic parser
+
platform identity

구조를 유지한다.

--------------------------------------------------

RAPUS는 단순 랜덤 상품코드를 사용하지 않는다.

현실 유통(EAN13 / 저울 바코드 / 물류 코드)을
semantic layer로 해석하여
플랫폼 상품 identity와 연결한다.

--------------------------------------------------

핵심 역할 분리:

scanCodeValue
=
현실 세계 바코드(EAN13)

productCode
=
RAPUS 플랫폼 공식 immutable identity

semanticProductCode
=
사람이 읽는 운영 semantic code

--------------------------------------------------

절대 원칙:

scanCodeValue
≠
productCode

semanticProductCode
≠
FK identity

semanticProductCode는
운영/물류/디버깅 semantic layer 전용이다.

SECTION 05 : SEMANTIC PRODUCT CODE
==================================================

semanticProductCode
=
사람이 읽는 운영 semantic code

--------------------------------------------------

예시:

NO379

--------------------------------------------------

의미:

NO
=
Normal Product

379
=
참외 품목번호

--------------------------------------------------

예시:

NO379
= 일반 참외

OR379
= 유기농 참외

PR379
= 프리미엄 참외

FR482
= 냉동 딸기

RF510
= 냉장 돼지고기

WT620
= 저울 생선

--------------------------------------------------

역할:

- 운영 표시
- 물류 semantic
- 관리자 검색
- 디버깅
- 저울상품 식별
- 운영자 코드 해석
- 재고 이동 추적
- 창고 운영
- 라벨 출력
- QR 출력 표시
- POS 상품 표시
- market feed semantic
- 행사상품 semantic
- 유통 상태 semantic
- 상품 상태 semantic

--------------------------------------------------

주의:

semanticProductCode는
실제 FK identity로 사용하지 않는다.

UNIQUE 제약 금지.

semanticProductCode는
운영 semantic layer로만 유지한다.

--------------------------------------------------

권장 타입:

VARCHAR(64)+

고정 길이 금지.
