Codex용 작업지시서(프롬프트)
- 정확한 파일 경로
- 수정 대상
- 생성/수정 파일 목록
- 금지사항
- 검증 기준
- 작업 완료 보고 형식

Claude용 지시서
- Codex용 작업지시서(프롬프트) +@
- 먼저 파일 구조를 읽어라
- 기존 코드 흐름을 요약하라
- 수정 계획을 먼저 제시하라
- 그 다음에 코드 수정하라
- 임의 생성 금지
- 변경 후 diff 관점으로 보고하라
=======================================
[필수 프로젝트 규칙]
1. 기존 파일 구조를 먼저 확인한다.

[통코드 / 부분수정 기준]
1. 기본 원칙은 1파일 = 1통코드이다.
2. 단, Codex 실제 작업에서는 수정량이 작고 기존 구조가 명확한 경우 부분수정 가능하다.
3. 부분수정 후 오류가 발생하면 즉시 해당 파일 전체를 1통코드 기준으로 정합 수정한다.
4. SECTION 단위 오류일 경우 해당 SECTION 전체를 수정 단위로 본다.
5. 최종 공유/검토/보고용 코드는 항상 파일 경로와 수정 범위를 명시한다.

[작업 시작 전 필수 확인]

Codex / Claude는 작업 전 반드시 다음을 확인한다.

1. 실제 파일 경로
2. 수정 대상 파일
3. 신규 생성 파일 여부
4. 삭제 파일 여부
5. DB/API/Service 수정 필요 여부
6. 금지사항 충돌 여부
7. 기존 UI/기능 보존 대상

[완료보고서 필수 형식]

1. 수정 파일 목록
2. 신규 생성 파일 목록
3. 삭제 파일 목록
4. 변경 요약
5. 유지한 기존 기능
6. 변경한 UI/State/Type
7. API 수정 여부
8. DB 수정 여부
9. Service/Controller 수정 여부
10. 금지사항 위반 여부
11. 검증 결과
12. 남은 이슈
13. 다음 작업 제안

[작업 범위 초과 처리]

Codex / Claude는 작업 중 지시서 범위를 초과하는 수정이 필요하다고 판단될 경우,
임의로 수정하지 말고 완료보고서의 "남은 이슈" 또는 "다음 작업 제안"에 기록한다.

작업 범위 밖의 DB/API/Service/라우팅/공통 레이아웃 수정은
별도 작업지시서 없이 진행하지 않는다.
