SECTION 03 : 통코드 수정 순서

// STEP 1 : 기존 통코드 확보
//   - 사용자가 전체 통코드(single file full code) 업로드
//   - AI는 기존 코드 누락, 경로, 섹션, UI/UX 구조 점검

// STEP 2 : 5WHY 오류 분석 / 원인 추적
//   - TypeScript, JSX, State, API, Backend 연결 등 오류 원인 파악
//   - UI/UX, 기능 영향 범위 검토

// STEP 3 : ADD 방식 코드 보충 / 수정
//   - 기존 로직(UI/UX) 유지 + 오류 수정 / UI 상태 반영 / 수동 fetch 버튼 추가
//   - 기존 코드 삭제, 축소, 교체 금지
     ※ 수정가능 절대요건 = 반드시 [수정요청]에만 기존 로직(UI/UX) 수정가능※

// STEP 4 : 라인 수 검증
//   - 기존 라인 대비 감소 → FAIL
//   - 기존 라인 대비 증가 → PASS
//   - 감소 발생 시 누락 판단 → 재생성 수행

// STEP 5 : 통코드 통합 검증
//   - UI/UX 영향 없음
//   - API 영향 없음
//   - State 구조 유지
//   - Routing 및 Effect 영향 없음

// STEP 6 : 전체 통코드 출력
//   - 단일 코드 블록으로 출력
//   - FILE PATH, 기존 라인 수, 수정 후 라인 수, 누락 없음 선언 포함
//   - 부분 코드, 예시 코드 출력 금지
